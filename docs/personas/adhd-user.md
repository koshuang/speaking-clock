# Persona: ADHD 用戶

## 基本資料

| 項目 | 內容 |
|------|------|
| 名稱 | 阿凱 |
| 年齡 | 25-35 歲 |
| 職業 | 軟體工程師 / 設計師 / 自由工作者 |
| 裝置 | 手機、電腦、平板（多裝置切換） |
| 特徵 | 注意力不足過動症 (ADHD)，已確診或自我認同 |

## 人物背景

阿凱是一位有 ADHD 特質的軟體工程師。他很聰明、有創意，在感興趣的事情上可以高度專注（hyperfocus），但日常生活的時間管理對他來說是一大挑戰。他常常忘記時間、拖延任務、被干擾分心。他試過很多效率工具，但大多太複雜或無法持續使用。他需要一個簡單、不需要動腦設定、有外部提醒的工具。

## ADHD 相關特徵

### 時間盲 (Time Blindness)
- 完全感受不到時間的流逝
- 常常抬頭發現已經過了 3 小時
- 對「5 分鐘」和「50 分鐘」的感受差不多

### 注意力特點
- **分心**：很容易被手機通知、突發想法打斷
- **過度專注**：一旦進入心流，會完全忘記其他事
- **轉換困難**：從一件事切換到另一件事很痛苦
- **難以開始**：知道該做，但就是無法開始

### 執行功能困難
- 難以估算任務所需時間
- 難以排列優先順序
- 容易忘記待辦事項
- 需要外部提示才能行動

### 情緒特點
- 對無聊的任務極度抗拒
- 時間壓力（deadline）才能產生動力
- 完不成事情會有強烈挫折感
- 對自己的「不自律」感到愧疚

## 一日作息（實際狀況）

| 計劃 | 實際狀況 |
|------|----------|
| 08:00 起床 | 按掉鬧鐘，滑手機到 9:00 |
| 09:00 開始工作 | 泡咖啡、收信、看新聞，11:00 才進入狀態 |
| 12:00 午餐 | 忘記吃飯，14:00 才想起來 |
| 14:00 專注工作 | 進入心流，忘記開會時間 |
| 18:00 下班 | 捨不得停，繼續做到 20:00 |
| 22:00 睡覺 | 躺在床上滑手機到凌晨 2:00 |

## 痛點 (Pain Points)

1. **忘記基本需求**
   - 忘記吃飯、喝水、上廁所
   - 過度專注時完全脫離現實

2. **無法準時**
   - 會議、約會總是遲到
   - 不知道準備需要多少時間

3. **任務啟動困難**
   - 拖延到 deadline 前一刻
   - 明知道該做，就是動不了

4. **Hyperfocus 失控**
   - 不該專注的事情專注太久
   - 錯過重要事項

5. **工具過載**
   - 試過太多 App，都用不久
   - 設定太複雜，懶得維護
   - 習慣用新鮮感消退後就放棄

6. **睡眠作息**
   - 晚上精神好、早上起不來
   - 沒有固定作息

## 需求 (Needs)

### 核心需求

| 優先級 | 需求 | 說明 |
|--------|------|------|
| P0 | 被動式提醒 | 不需要我記得看，它會叫我 |
| P0 | 時間錨點 | 定時報時讓我知道時間在流逝 |
| P1 | 零設定啟動 | 打開就能用，不需要複雜設定 |
| P1 | 打斷 hyperfocus | 幫我從過度專注中抽離 |
| P2 | 彈性計時 | 不要強迫我遵守固定時間，有彈性 |
| P2 | 簡單待辦 | 記住我的待辦，但不要太多功能 |

### 功能需求對應

| 需求 | 現有功能 | 缺少/待改進 |
|------|----------|-------------|
| 被動式提醒 | ✅ 語音報時 | - |
| 時間錨點 | ✅ 定時報時（1/5/10/15/30/60 分） | 可增加「到整點報時」選項 |
| 零設定啟動 | ⚠️ 部分 | 需要更簡化的首次設定 |
| 打斷 hyperfocus | ✅ 報時 + 任務提醒 | 可增加「強制提醒」模式 |
| 彈性計時 | ⚠️ 固定時長 | 可增加「延長」「暫停」選項 |
| 簡單待辦 | ✅ 待辦清單 | 已經夠簡單 |

## 使用情境 (Use Cases)

### 情境 1：工作日時間錨點
> 阿凱設定每 30 分鐘報時一次。在過度專注時，語音報時「現在是下午 3 點 30 分」讓他驚覺已經過了 2 小時。他可以決定繼續或休息，但至少有意識。

### 情境 2：基本需求提醒
> 阿凱設定待辦：「12:30 吃午餐」「15:00 喝水+伸展」「18:00 下班」。報時器在這些時間語音提醒他基本生理需求。

### 情境 3：會議不遲到
> 下午 2 點有視訊會議。阿凱設定「13:50 準備開會」的提醒。報時器會在 10 分鐘前提醒他結束手上的事、準備開會。

### 情境 4：Pomodoro 變形
> 阿凱用任務計時器設定「專注 45 分鐘」。時間到時報時器會打斷他，提醒休息。他可以選擇「再 15 分鐘」或「休息 10 分鐘」。

### 情境 5：睡前提醒
> 晚上 11 點，報時器播報「現在是晚上 11 點，該準備睡覺了」。阿凱可能會忽略，但至少有一個外部提示。

## 與一般時間管理工具的差異

### 為什麼其他工具不適合

| 工具類型 | 問題 |
|----------|------|
| 行事曆 | 需要主動看，不會主動提醒 |
| 複雜待辦 App | 設定太多，維護成本高 |
| 番茄鐘 | 太死板，不符合 ADHD 節奏 |
| 手機鬧鐘 | 只有聲音，沒有語音內容 |
| 智慧手錶 | 震動容易忽略 |

### 語音報時器的優勢
1. **被動式**：自動播報，不需要我看
2. **語音內容**：不只是鈴聲，是有意義的資訊
3. **簡單**：功能剛好，不會過載
4. **持續存在**：不是一次性鬧鐘，是持續的時間錨點

## 設計建議

### 介面設計
- 極簡風格，減少視覺干擾
- 最重要功能一鍵可達
- 減少設定選項，提供智慧預設
- 深色模式（夜間使用）

### 語音設計
- 語調平和不刺耳
- 可調整音量、語速
- 提供「溫和提醒」和「強力提醒」選項
- 可自訂提醒語句

### 互動設計
- 一鍵開始報時（無需設定）
- 「再 5 分鐘」「再 15 分鐘」延長選項
- 不要有懲罰機制（會造成挫折）
- 減少決策：「推薦設定」一鍵套用

### 特殊功能建議
- **漸進式提醒**：重要事項前 30/15/5 分鐘各提醒一次
- **body doubling 模式**：每 10 分鐘報時，模擬有人陪伴
- **時間預估輔助**：任務結束後顯示「實際花費 45 分鐘」
- **防 hyperfocus**：連續工作 90 分鐘強制語音提醒

## 與其他 Persona 的關係

| Persona | 共通點 | 差異點 |
|---------|--------|--------|
| 小學生 | 需要外部提醒 | 成人自主管理，不需家長監督 |
| 家長 | - | 管理對象是自己，不是孩子 |
| 遠距工作者 | 時間管理需求 | ADHD 有額外的認知困難 |
| 視障使用者 | 依賴語音 | 視力正常，但需要聽覺提示 |

## 設計哲學

> **「不要試圖修復我，幫助我找到方法。」**

好的 ADHD 友善設計：
- ✅ 接受使用者的認知特點
- ✅ 減少認知負擔，不增加
- ✅ 彈性而非強制
- ✅ 外部化記憶和時間感
- ✅ 正向、不批判

壞的設計：
- ❌ 強調「自律」「堅持」
- ❌ 複雜的設定和維護
- ❌ 懲罰機制（扣分、警告）
- ❌ 假設使用者會記得查看
